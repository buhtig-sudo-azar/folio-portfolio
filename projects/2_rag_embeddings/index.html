В основе технологии RAG (Retrieval-Augmented Generation) лежит способность превращать текст в эмбеддинг — вектор из 1024 чисел, описывающий смысл высказывания. Этот проект представляет собой исследовательский стенд, визуализирующий фундаментальный механизм: как текст становится координатой на смысловой карте и как по этим координатам работает поиск.

Основная цель стенда — продемонстрировать, что модель не использует метки, не обращается к API и не полагается на ключевые слова. Она оперирует исключительно математикой смысла, извлечённой из самого текста.

Природа эмбеддинга

Эмбеддинг не является случайным набором чисел. Это проекция текста в 1024-мерное пространство, где каждое измерение отвечает за определённый аспект контекста: тематику (IT/еда), тип объекта (сервер/софт), степень технической глубины и сотни других характеристик.

В распределённом смысловом пространстве действует фундаментальный принцип: похожие тексты → близкие координаты, разные тексты → далёкие координаты. Для измерения близости используется косинусное сходство — угол между векторами, игнорирующий длину текста.

Важнейший аспект: модель сама выделила аспекты смысла в процессе обучения. Человек не размечал документы как «IT» или «еда». Модель миллионы раз видела пары похожих и непохожих текстов и самостоятельно настроила внутренние веса так, что похожие тексты получают похожие координаты.

Сравнение с классическим подходом на метках

  Классический подход (метки) Эмбеддинги
Кто размечает Человек вручную Модель сама
Плавные переходы Невозможны (документ ИЛИ IT, ИЛИ еда) Естественны (70% IT, 30% еда)
Масштабирование Требует переразметки при новых темах Автоматически
Поиск По ключевым словам По смыслу

Стенд наглядно показывает это различие: запрос «запустить сайт на сервере» находит Nginx, хотя в запросе нет ни слова «nginx», ни слова «веб-сервер». Модель работает со смыслом, а не с лексикой.

Лабораторная среда

В рамках демо-приложения реализован интерактивный терминал, воспроизводящий работу настоящего RAG-конвейера:

  • Загрузка модели: intfloat/multilingual-e5-large — 1024-мерные векторы, поддержка 100+ языков включая русский.
  • Индексация документов: 5 тестовых текстов превращаются в векторы и сохраняются в виртуальной векторной базе.
  • Косинусное сравнение: попарное сравнение всех документов с визуализацией шкалы сходства.
  • Семантический поиск: обработка запросов пользователя и ранжирование документов по смыслу.

Пользователь может выполнять команды напрямую в терминале:

  • demo — полный прогон: модель, векторы, сравнение всех пар, поиск по 3 запросам с аналитикой результатов.
  • search ЗАПРОС — семантический поиск по тестовой коллекции.
  • what — детальное объяснение механики эмбеддингов.
  • rag — визуальная схема трёхэтапного конвейера (Index → Retrieve → Generate).
  • models — сравнение текущей модели с устаревшей (all-MiniLM-L6-v2, только английский).

Экспериментальные данные

Стенд работает с пятью документами: Nginx, Apache, Docker, systemd и рецепт борща. Результаты поиска демонстрируют чёткое разделение IT и нетехнического контента:

Запрос Топ-1 Сходство
«веб-сервер для статических файлов» Apache 0.810
«изолированная среда для тестирования» Docker 0.803
«что приготовить из свёклы» Борщ 0.769

Показательно, что IT-документы имеют взаимное сходство 0.84–0.89 (высокое), а сходство IT↔борщ не превышает 0.75. Разрыв в 0.10–0.15 достаточен для надёжного разделения.

Техническая реализация и стек

  • Модель эмбеддингов: intfloat/multilingual-e5-large (~500 МБ), работает локально, без API-запросов.
  • Вычисления: Python, sentence-transformers, scikit-learn (cosine_similarity), numpy.
  • Интерфейс: jQuery Terminal — эмулятор терминала с кастомными командами.
  • Деплой: статический HTML на Vercel, не требует бэкенда.
